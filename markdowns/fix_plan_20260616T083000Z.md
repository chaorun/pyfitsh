### Bug：inaps 共享导致 double-free + 与 CLI 原版不一致

**问题 1：** 第 1810-1825 行构建 photstar 时，所有 star 共享同一个全局 `inaps` 指针（`p->inaps = inaps`），但 `_fiphot_free_photstar` 对每个 star 各 free 一次 → **double-free**。

**问题 2（根因）：** CLI 原版 `read_raw_photometry`（`fiphot-io.c:327-337`）为**每个 star 独立 malloc** inaps，且 `do_subtracted_photometry`（`fiphot.c:888-889`）内部支持两种模式：
```c
if (inaps != NULL)  aps=inaps,n=numap;
else                aps=ps[i].inaps,n=ps[i].n;  // 每星独立
```
当传入全局 inaps=NULL 时，函数自动使用每星独立的 `ps[i].inaps`。Cython 版本应该模仿 CLI。

| CLI 原版 `read_raw_photometry` | Cython 新函数（修复后） |
|---|---|
| `ps[np].inaps = malloc(apgeom*n)` | `p->inaps = malloc(apgeom*nap)` |
| `ps[np].ninap = n` | `p->ninap = nap` |
| `do_subtracted_photometry(..., inaps, ninap, ...)` — inaps 可能为 NULL | `do_subtracted_photometry(..., NULL, nap, ...)` |
| 函数内部 `if(inaps!=NULL) aps=inaps; else aps=ps[i].inaps` | 函数内部自动使用每星 `ps[i].inaps` |
| 逐星 free inaps（无共享问题） | 逐星 free inaps（无共享问题） |
