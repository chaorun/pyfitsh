## 文件 1：`fitsh_cython/fiphot/fiphot_core.c`

### 存档 vs 当前

- **存档版本**：1726 行，末尾为 `fiphot_magnitude_fit_flat` 函数的结束 `}`
- **当前版本**：1805 行，在 1726 行后新增：
  - 第 1727-1733 行：注释掉的 dump 代码块（5 行，无影响）
  - 第 1735-1805 行：`read_raw_photometry_cy` 函数（71 行，逐字段模仿 CLI `read_raw_photometry`）

### Bug（已修复）

旧版 `fiphot_photometry_from_raw_cython` 存在两个问题：
1. 所有 star 共享同一个 `inaps` 指针 → `_fiphot_free_photstar` 逐星 free → **double-free**
2. 与 CLI 原版不一致（CLI 每星独立 malloc inaps，通过 `aperture_definition_parse` 构建）

已在本版中**重写为 `read_raw_photometry_cy`**，仅做 photstar 构建（对应 CLI `read_raw_photometry` 的单一职责），采用每星独立 inaps + rfflux，ref_mag 动态控制 use_ref。

### 新函数签名

```c
int read_raw_photometry_cy(
    photstar **rps, int *rnp,
    double *raw_data, int nstar, int nap,
    double *ref_mag_arr, double *ref_col_arr, double *ref_err_arr)
```

详见 `read_raw_photometry.md`。
