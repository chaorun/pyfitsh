# Cython C 管线集成经验总结

## 核心原则

**原版代码逐字复制，不要重写、不要合并函数、不要省略流程、不要提前优化。**

---

## 一、常见错误模式

### 1. 擅自跳过函数（最致命的错误）

**表现：** 觉得某个函数"当前用不到"就不搬。

**案例：**
- ficonv: 跳过 `kernel_init_images()` → `k->image` 为 NULL，`convolve_point` 应 segfault
- ficonv: 跳过 `fit_kernels()` / `fit_kernels_native()` → 缺失 mask 合并、迭代拟合、加权
- fistar: 跳过 `fits_mask_mark_nans()` → NaN 像素未被 mask
- fistar: 跳过 `star-cand-pp/trb/biq.c` → 缺算法支持

**原则：** `main()` 里调用的每一行流程函数都要搬，不确定的宁可多搬不少搬。

### 2. 擅自修改参数（隐性优化）

**表现：** 觉得某个参数"用默认值就行"就硬编码。

**案例：**
- ficonv: `bdc=1` 替代 `bdc=32` → 改变了矩阵组装的分块策略
- fistar: `SHAPE_ELLIPTIC` 替代 `SHAPE_GAUSS` → 星拟合结果不同
- fistar: `ALG_LNK` 硬编码 → 缺算法切换

**原则：** 所有原版 `main()` 中由 `scanarg` 解析的参数，都要作为管线入口参数传入。

### 3. 数据结构别名（自引用错误）

**案例：**
- ficonv: `xl.kernels = kl.kernels; xl.nkernel = kl.nkernel` → subblock 内 realloc 自拷贝
- 原版: `xlist->nkernel=0; xlist->kernels=NULL` → 正确语义

**原则：** 逐行对照原版的初始化逻辑，不要自作主张"简化"。

### 4. 缺少必要的 mask 处理

**案例：**
- ficonv: 卷积时 mask 全零 → 边缘像素越界访问
- 原版: `mask_conv = fits_mask_expand_false(mask, sx, sy, hsize, -1, -1, 1)` → 扩展 kernel 边缘

**原则：** mask 创建、合并、扩展每一步都要和原版一致。

## 二、编译链问题

### 5. fits 库依赖链

**问题：** 原版代码广泛使用 `fits_image_duplicate/free`、`fits_create`、`fits_headerset_*` 等。编译整个 fits 库（8 个 .c）会带来几百 KB 的膨胀和大量内部依赖。

**解决：** 用 `fits_stubs.c` 提供空实现。关键是要检查哪些 fits 函数**确实被运行时调用**（fits_image_duplicate/free 仅在 collective fit 分支，管线不走），哪些仅在**编译期需要符号**。

### 6. 多管线共享符号冲突

**问题：** ficonv_pipeline.c 和 fistar_pipeline.c 都定义了 `fprint_error`、`fprint_warning`、`is_verbose`、`progbasename` → `duplicate symbol`。

**解决：** 第一个搬入的管线定义全局符号，后续管线用 `extern` 引用。

### 7. Cython .pxd 声明与 .h 不一致

**问题：** 修改了 C 函数签名但只更新 .c/.h，忘记同步 .pxd → Cython 编译错误或运行时参数错位。

**解决：** 改函数签名时三处同步更新：头文件 (.h)、实现 (.c)、声明 (.pxd)。

### 8. Cython 语法限制

- `.pyx` 里 `cdef` 方法内不能定义 `def` 内嵌函数
- 不能在 dict literal 中使用条件表达式（`if ... else`）
- struct 成员的 `double *` 字段需要先赋值给局部 `cdef` 变量再访问

## 三、对比验证

### 9. 参考输出必须来自相同参数

**问题：** 用不同格式参数生成的 C 二进制输出做对比 → 列索引错误。

**案例：**
- ficonv: dump 文件有 8 字节头 (sx, sy)，直接用 `np.fromfile` 会偏移
- fistar: `mom` 是多值字段，会偏移后续所有列的索引

**解决：**
- Dump 文件要按格式逐字节解析（跳过 header）
- 文本输出要去掉 `mom` 等多值字段再用固定列索引对比

### 10. 排序 vs 最近邻匹配

**问题：** C 二进制按 `-s x` 排序，Cython 不排序。直接比较 `array[0]` 会对比不同星。

**解决：** 用 `cKDTree` 按位置 (x, y) 做最近邻匹配，确保同一颗星才比较。

### 11. 浮点精度判断阈值

**问题：** `np.allclose(rtol=1e-10)` 在 C 二进制只输出 6 位有效数字时必然失败。

**解决：** C 二进制输出精度约 1e-3，用 `max_diff < 0.01` 做判断。数值相等性用科学计数法比较。

## 四、集成步骤（推荐流程）

### Step 1: 只读阶段（不动代码）

1. 在 `origincode/` 找到原版 `main()`，逐行列出所有函数调用
2. 列出每个函数的依赖：include 头、外部 .c 文件
3. 用 `diff` 验证文件是否逐字节一致
4. 生成原版 dump 输出作为参考基线

### Step 2: C 二进制编译

1. 复制所有 .c/.h 到 testgrmatch
2. 单行 gcc 命令编译（不修改 Makefile）
3. 运行并与系统版输出对比，确认可重现

### Step 3: Cython 集成

1. 复制 .c/.h 到 fitsh_cython/（保持目录结构）
2. 创建 `xxx_pipeline.c`：逐字搬入管线函数 + 新入口函数
3. 入口函数只做三件事：
   - flat 数组 → fitsimage/char** 行指针（不分配新内存）
   - 调管线函数（逐字同原版）
   - 提取结果到 flat 数组
4. 更新 setup.py sources、include_dirs
5. 更新 .pxd（声明）和 .pyx（Python wrapper）
6. 编译 → 运行 → 与 dump 逐字段对比

### Step 4: 清理

1. 检查未定义符号（`nm -u`），建 stub 替代死代码
2. 删除不用的 .c 文件
3. 删除调试 print

## 五、教训速查

| 错误 | 表现 | 修复 |
|------|------|------|
| 跳过 kernel_init_images | k->image=NULL，convolve_point 崩溃 | 加上调用 |
| xlist 别名 | realloc 自拷贝 | 改为 nkernel=0, kernels=NULL |
| bdc=1 vs bdc=32 | 矩阵组装策略变化 | 传入原版参数 |
| 缺少 mask 扩展 | 边缘像素越界 | 加 fits_mask_expand_false |
| 卷积掩码未复制 | Python 侧拿到空 mask | memcpy 输出 mask |
| 不包含 fits/mask 依赖 | 编译/链接报错 | 搬对应 .c 或用 stub |
| Cython .pxd 未同步 | 签名不匹配 | 三处同步更新 |
| 用排序比较而非匹配 | 对比了不同星 | cKDTree 最近邻匹配 |
| fits_stubs.c 缺符号 | 链接失败 | nm -u 列出全部，补齐实现 |
